Easily Create New Digital Garden Post and Projects Blog

Functions to quickly create a new blog post. This depends on org-hugo and yas-snippet.

(defvar mein-site-dir "~/org/website/mein-site")

(defun rz/blog-new-garden ()
 "Create a new `org' file for sharing ideas in your blog's digital garden."
 (interactive)
 (let* ((blog-garden-dir (concat mein-site-dir "/content-org/garden"))
        (blog-garden-file (read-from-minibuffer "What is the name of your garden file \(use kebab-case\):")))
  (split-window-right)
  (find-file (concat blog-garden-dir "/" blog-garden-file ".org"))
  (insert (yas-expand-snippet (yas-lookup-snippet "hugogarden")))
  (save-buffer)
  (message "Happy sharing your garden of ideas, Reza!")))

(defun rz/blog-new-project ()
 "Create a new `org' file for sharing project in your blog."
 (interactive)
 (let* ((blog-projects-dir (concat mein-site-dir "/content-org/projects"))
        (blog-projects-file (read-from-minibuffer "What is the name of your project file \(use kebab-case\):")))
  (split-window-right)
  (find-file (concat blog-projects-dir "/" blog-projects-file ".org"))
  (insert (yas-expand-snippet (yas-lookup-snippet "hugoproject")))
  (save-buffer)
  (message "Happy sharing your project, Reza!")))

Easily write Book Review

I have my book collection managed with Calibre and Zotero. Those two program produced nice BibTeX files for my Zettelkasten set up in Org-Roam, handled by org-roam-bibtex. In such everything in my library is neatly structured and stored. So why would I need to write everything from scratch and looking for cover manually? I won’t. Here is the function that make that magic happens (As with all of my Emacs setting you could see that in my doom-emacs config repo):

(defun rz/blog-book-review (citekey)
  "Function to create a new blog file for book review. Handles
folder creation and book cover copy for a seamless book
reviewing. Decided to not make this `interactive' as it could
only be executed through `orb-note-action'. NOTE: `citekey' is only
passed but not used just because it is automatically passed by
`orb-note-action'"

  ;; Variable Definition
  (let* ((book-review-org-dir (concat mein-site-dir "/content-org/library"))
         (book-review-md-dir (concat mein-site-dir "/content/library"))
         (book-calibre-id (file-name-sans-extension (file-name-nondirectory buffer-file-name)))
         (book-review-org-id-dir (concat book-review-org-dir "/" book-calibre-id))
         (book-review-md-id-dir (concat book-review-md-dir "/" book-calibre-id))
         (cover-from (save-excursion
                         (goto-char (point-min))
                        ;; Since my cover link are within the `Cover' header
                         (when (re-search-forward "^\\*\\s-+Cover\\s-*$" nil t)
                          (org-next-link)
                          (org-element-property :path (org-element-context)))))
         (cover-to (concat book-review-md-id-dir "/featured.jpg")))

    ;; create the folder if it is not exists
    (unless (file-exists-p book-review-org-dir)
      (make-directory book-review-org-dir t))
    (unless (file-exists-p book-review-md-dir)
      (make-directory book-review-md-dir t))
    (unless (file-exists-p book-review-md-id-dir)
      (make-directory book-review-md-id-dir t))
    (unless (file-exists-p book-review-org-id-dir)
      (make-directory book-review-org-id-dir t))
    ;; copy the cover from calibre if it is not already
    (unless (file-exists-p cover-to)
      (copy-file cover-from cover-to))

    ;; also create a new split buffer for the whole thing while keeping the old one
    ;; opened to allow easier review
    (split-window-right)
    (find-file (concat book-review-org-id-dir "/index.org"))
    ;; only insert snippet in new file
    (if (= (buffer-size) 0)
        (insert (yas-expand-snippet (yas-lookup-snippet "hugolib"))))
    (save-buffer)
    (message "Happy sharing your book review, Reza!")))

Also make sure to have hugolib snippet in yas-snippet

# -*- mode: snippet -*-
# name: hugolib
# uuid: hugolib
# key: hugolib
# condition: t
# --
#+TITLE: ${1:Book title}
#+AUTHOR: ${2:Book author}
#+date: `(format-time-string "%Y-%m-%d")`
#+STARTUP: content
#+HUGO_SECTION: library/`(file-name-nondirectory (directory-file-name (file-name-directory buffer-file-name)))`
#+HUGO_DRAFT: true
#+HUGO_CUSTOM_FRONT_MATTER: :summary "${3:Summary}" :library_tags '(${4:Tags}) :status seeding

`%`$0

Instead of thinking about where to put the new keybinding, I decided to just add it into org-roam-bibtex as this is where I am most likely start building up on the idea to write a book review.

(add-to-list 'orb-note-actions-user '("Share a review of this book in blog" . rz/blog-book-review))
